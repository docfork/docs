**Cabinets** are Docfork's primary "Context" mechanism. They function like an `.env` file for your documentationâ€”locking your AI agent to a specific set of libraries to prevent hallucinations from unrelated frameworks.

## How Cabinets Work

Instead of searching the entire index of 10,000+ libraries, a Cabinet limits the search scope to only the libraries you manually add to it.

<Steps>
  <Step>
    ### Create
    Go to the [Dashboard](https://app.docfork.com) and create a Cabinet named after your project (e.g., `general`).
  </Step>
  <Step>
    ### Populate
    Add the libraries you use: 
    - `stripe/stripe-node` 
    - `react-hook-form/react-hook-form`
    - `shadcn-ui/ui`
  </Step>
  <Step>
    ### Connect
    Add the `DOCFORK_CABINET` header to your MCP configuration.
  </Step>
</Steps>

## Configuration

**VS Code / Cursor Config:**

```json
{
  "docfork": {
    "url": "https://mcp.docfork.com/mcp",
    "headers": {
      "DOCFORK_API_KEY": "YOUR_API_KEY",
      "DOCFORK_CABINET": "general"
    }
  }
}
```
